# 2026-05-30

## 前端 WebSocket / API 连接问题修复
- **问题**：`vite.config.js` proxy target 指向 `rpi5.lan:8000`，但 `.env` 中 `VITE_API_URL` 直接设了 `http://rpi5.lan:8000/api`，导致 axios 绕过 Vite proxy 直接请求树莓派，触发 CORS 错误。
- **修复**：
  - `frontend/vite.config.js`：proxy target 改为 `rpi5.lan`（通过 mDNS 可达）
  - `frontend/.env`：注释掉 `VITE_API_URL` 和 `VITE_WS_URL`，让请求走 Vite proxy
  - `frontend/src/utils/api.js`：修复 `connectWebSocket`，开发模式下使用 `window.location` 构造 WS URL
  - `backend-go/cmd/remoteid/main.go`：抓包启动失败时不再 `os.Exit(1)`，仅 warn 并继续运行 API

## Standard / Source 字段为空修复
- **问题**：`processor.go` 中 `updateDroneFromMessage` 第 352 行从 `msg.Data["standard"]` 取值，但 parser 把 standard 放在 `msg.Standard` 结构体字段上。
- **修复**：
  - `backend-go/pkg/types/models.go`：在 `DroneData` 中新增 `Source` 字段
  - `backend-go/internal/drone/processor.go`：`drone.Standard = msg.Standard`，`drone.Source = msg.Source`
  - 前端 `DroneDetail.vue` 和 `App.vue`：展示 Standard 和 Source 字段

## 信息展示不完整修复
- **问题**：`App.vue` 中 `selectDrone` 函数把详情 API 数据存到 `selectedDroneDetail`，但模板渲染用 `selectedDrone`（只含列表数据），导致 info block 不显示详情字段。
- **修复**：`selectDrone` 中将详情数据合并到 `selectedDrone`：`selectedDrone.value = { ...drone, ...detail }`
- 新增 Flight Status / Height Type / Vert Speed 展示
- 新增 source 列到侧边栏可选列

## 部署方式
- 后端运行在树莓派 5（`rpi5.lan`，IP: `192.168.6.220`）
- 前端 `npm run dev` 在 Mac 开发机 `127.0.0.1:8080`
- 通过 Vite proxy 转发 `/api` 和 `/ws` 到 `rpi5.lan:8000`
- 后端需要在树莓派上编译部署（因为 gopacket/pcap 依赖 CGO）
